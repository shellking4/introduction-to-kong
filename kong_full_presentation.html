ğŸ¦ What is Kong?

Definition

Kong is a cloud-native, platform-agnostic, scalable API Gateway distinguished for its high performance and extensibility via plugins.

Technical Foundation

  • Built on Nginx and OpenResty (lua-nginx-module)
  • Written in Lua for high performance
  • Open-source with enterprise offerings
  • Cloud-native and Kubernetes-ready
ğŸ¯ Key Value: Single entry point for all API traffic with centralized control for security, monitoring, and management.

ğŸ”§ Core Concepts: Services & Routes

1ï¸âƒ£ Service

A Service wraps properties of HTTP(s) endpoints that accomplish specific tasks, including upstream URLs for proxying requests.

# Create a service curl -i -X POST http://localhost:8001/services \ --data "name=user-service" \ --data "url=http://user-api:8080"

2ï¸âƒ£ Route

Routes define how incoming requests map to services based on paths, headers, methods, etc.

# Create a route for the service curl -i -X POST http://localhost:8001/services/user-service/routes \ --data "paths[]=/api/users" \ --data "methods[]=GET" \ --data "methods[]=POST"

Now: http://kong:8000/api/users â†’ http://user-api:8080/api/users

ğŸ”§ Core Concepts: Consumers & Plugins

3ï¸âƒ£ Consumer

Wraps properties of anyone using API endpoints. Used for tracking, access control, and rate limiting.

# Create a consumer curl -i -X POST http://localhost:8001/consumers \ --data "username=john_doe" # Add API key for consumer curl -i -X POST http://localhost:8001/consumers/john_doe/key-auth \ --data "key=secret-api-key-123"

4ï¸âƒ£ Plugin

Pluggable features to enrich functionalities during request and response lifecycle.

# Enable JWT authentication plugin curl -X POST http://localhost:8001/services/user-service/plugins \ --data "name=jwt"

Example: Adding JWT Authentication

# Step 1: Enable JWT plugin on service curl -X POST http://localhost:8001/services/user-service/plugins \ --data "name=jwt" # Step 2: Create consumer with credentials curl -X POST http://localhost:8001/consumers \ --data "username=mobile_app" curl -X POST http://localhost:8001/consumers/mobile_app/jwt \ --data "key=my_jwt_key" \ --data "secret=my_secret" # Step 3: Now all requests require valid JWT token!

âš¡ Key Feature: Rate Limiting

Protect Your APIs from Abuse

Kong can apply rate limiting rules by:

  • IP Address
  • Consumer/API Key
  • Service
  • Header value

Example: 100 Requests per Minute

curl -X POST http://localhost:8001/services/user-service/plugins \ --data "name=rate-limiting" \ --data "config.minute=100" \ --data "config.hour=5000" \ --data "config.policy=local"
âš ï¸ Important: When limit is exceeded, Kong returns HTTP 429 (Too Many Requests) with retry-after headers.

Advanced: Different Limits per Consumer Tier

  • Free tier: 100 req/min
  • Premium tier: 1000 req/min
  • Enterprise tier: Unlimited
